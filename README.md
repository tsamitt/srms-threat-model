# SRMS Threat Model

## Overview

This project applies basic threat modeling concepts to the Smart Service Request Management System (SRMS). The goal is to identify possible threats to the system and document practical mitigation strategies.

Threat modeling is useful because it helps security concerns get addressed early in the system design process rather than after deployment.

## Objectives

- identify possible threats to the SRMS application
- review attack scenarios affecting confidentiality, integrity, and availability
- recommend practical mitigation strategies
- strengthen security thinking during system design

## Skills Demonstrated

- threat modeling
- security analysis
- secure design thinking
- application risk documentation

## Files in This Repository

- `threat-model.md` - Main threat model overview
- `attack-scenarios.md` - Example threat scenarios
- `mitigation-strategies.md` - Recommended security controls

## What I Learned

This project helped me think about application security earlier in the design process and understand how common threats can affect web-based systems.
